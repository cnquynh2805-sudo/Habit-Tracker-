# Architecture Decision: Backend-as-a-Service (BaaS) Selection

**Date**: 2026-06-16
**Status**: Recommended / Accepted
**Decision**: Migrate to **Xano** as the primary Backend-as-a-Service (BaaS) provider.

## Context and Problem
For a highly structured application where entities (`Habits`, `Checkins`, `Goals`) are deeply coupled and dependent, we need a system that supports strong relational integrity. Alternative solutions like Appwrite (document-based) or Supabase (SQL-heavy) require either simulating relationships or configuring complex manual Row Level Security (RLS) which introduces a "nightmare" of configuration overhead for an MVP.

We require a structured, cloud-based, free, 100% RESTful BaaS with a simplified security model.

## Why Xano is the Superior Choice

1. **Native Relational Integrity**
   - Xano is built on a true relational database engine. We can create strict "Foreign Key" relationships between Habits, Checkins, and Goals visually, matching our exact OpenAPI structural dependencies.

2. **100% RESTful by Default**
   - Every database table automatically generates a full set of CRUD REST API endpoints (`GET`, `POST`, `PATCH`, `DELETE`). No custom backend code or CLI tools are required.

3. **One-Time Security Setup**
   - Security is defined at the API endpoint level via a simple "Add-on" logic. By checking "Requires Authentication" on an endpoint, it is protected globally, bypassing the need for complex database row-level security configurations.

4. **API Transforms for Coupled Entities**
   - Xano's visual logic builder allows us to set up "API Transforms". When the frontend requests a Habit object, the endpoint can be configured to automatically bundle associated Checkins and Goals in a single response, solving our highly-coupled data problem without writing complex SQL joins.

5. **Permanent Free Cloud Tier**
   - Provides relational database hosting, the REST API builder, and cloud infrastructure on a hobbyist tier perfectly suited for this project.

6. **Agentic Workflow Compatibility**
   - **Schema Generation**: Agents can parse our `openapi.yaml` to define the relational tables and foreign keys in Xano.
   - **REST Mapping**: Agents can map OpenAPI routes directly to Xano's auto-generated endpoints and build the TypeScript service layers.

## Comparison Summary
| Feature | Xano | Appwrite |
|---------|------|----------|
| **Relational Data** | Native (True Foreign Keys) | Document-based (Simulated) |
| **API Endpoints** | Autogenerated for every table | Autogenerated |
| **Security Setup** | One-click per endpoint | Dashboard-level checkboxes |
| **Frontend Integration** | 100% RESTful | 100% RESTful |

## Conclusion
By choosing Xano, we achieve the strict relational structure the Habit Tracker demands alongside a "vibe-coding" friendly interface that eliminates the need to configure complex database security or write custom backend code.
